PostgreSQL History

INGRES

INteractive Graphics REtrevial System

Started by Berkley in 1970s

Spawned the development of Microsoft SQL server

POSTGRES

POST (IN)GRES

Started by Berkley in 1986 with funding from DARPA

Design goal was to allow the creation of custom types, which was not possible in Ingres

Made relationships a first class component of the data model using rules

POSTGRES95

SQL interpreter was added in 1995

Decided that was a dumb name and started calling it PostgreSQL

Data Organization
Cluster - manages data, manages users
Database - All data is isolated to a specific database
Schema - Data can be shared accross schema, useful for namespacing
Types

Numeric

smallint 2 bytes small-range integer -32768 to +32767
integer 4 bytes typical choice for integer -2147483648 to +2147483647
bigint 8 bytes large-range integer -9223372036854775808 to +9223372036854775807
decimal or numeric variable user-specified precision, exact up to 131072 digits before decimal; up to 16383 digits after decimal
real 4 bytes variable-precision, inexact 6 decimal digits precision
double precision 8 bytes variable-precision, inexact 15 decimal digits precision
smallserial 2 bytes small autoincrementing integer 1 to 32767
serial 4 bytes autoincrementing integer 1 to 2147483647
bigserial 8 bytes large autoincrementing integer 1 to 9223372036854775807
You must be careful when casting money to another type because you can lose precision

Monetary

Yup, there is a money type.. just be careful

money 8 bytes currency amount -92233720368547758.08 to +92233720368547758.07

Character

character varying(n), varchar(n) variable-length with limit
character(n), char(n) fixed-length, blank padded
text variable unlimited length
The first few bytes are used as an escape character

Binary

bytea 1 or 4 bytes plus the binary string variable length

Date

Supports just about every input format that you can think of
timestamp [ (p) ] [ without time zone ] 8 bytes 4713 BC 294276 AD
timestamp [ (p) ] with time zone 8 bytes 4713 BC 294276 AD
date 4 bytes (1 day precision) 4713 BC 5874897 AD
time [ (p) ] [ without time zone ] 8 bytes 00:00:00 24:00:00
time [ (p) ] with time zone 12 bytes 00:00:00+1459 24:00:00-1459
interval [ fields ] [ (p) ] 16 bytes -178000000 years 178000000 years

Boolean

Uses 1 byte

true false
1 0
yes no
on off

Enumerated

              CREATE TYPE day AS ENUM ('sunday', 'monday', 'tuesday', ...);
            
              SELECT * FROM orders WHERE order_day > 'tuesday';

Geometric

point 16 bytes Point on a plane (x,y)
line 32 bytes Infinite line {A,B,C}
lseg 32 bytes Finite line segment ((x1,y1),(x2,y2))
box 32 bytes Rectangular box ((x1,y1),(x2,y2))
path 16+16n bytes Closed path (similar to polygon) ((x1,y1),...)
path 16+16n bytes Open path [(x1,y1),...]
polygon 40+16n bytes Polygon (similar to closed path) ((x1,y1),...)
circle 24 bytes Circle <(x,y),r> (center point and radius)

Network Address

cidr 7 or 19 bytes IPv4 and IPv6 networks
inet 7 or 19 bytes IPv4 and IPv6 hosts and networks
macaddr 6 bytes MAC addresses
